Colour is not very useful to us as it changes with orientation and lighting. We more care about edges

0D - featureless
1D - edges
2D - corners - has a desirable invariance to lighting

**1D edge detection**
We must smooth our signal, then differentiate it.
We will be smoothing by convolving with the Gaussian kernel: $g_\sigma(x)=\frac{1}{\sigma\sqrt{2\pi}}exp(-\frac{x^2}{2\sigma^2})$

Then take the derivative. As it is a linear operation, we could do it in either order
$s'(x)=\frac{d}{dx}[g_{\sigma}(x)*I(x)]=g_{\sigma}'*I(x)$

Interpolation can be used to find the maxima. Or we can take the derivative again and find where we get a zero crossing.
A smaller $\sigma$ brings out all the edges, a larger one only brings out the larger ones

**2D edge detection**
We do a 2D convolution with the 2D Gaussian $G_{\sigma}(x,y)$
$S(x,y)=\int_{-\infty}^{\infty}\int_{-\infty}^{\infty}G_{\sigma}(u,v)I(x-u,y-v)dudv$

The next step would be to take the gradient using the grad function $\nabla S = \nabla (G_{\sigma}*I)$
The next step is non-maximal suppression. Edge elements, or edgels are placed at locations where $|\nabla S|$ is greater than local values of $|\nabla S|$ in the directions $\pm \nabla S$.
Next all the edgels are thresholded, so only that with a magnitude over a certain value are kept.

In practice, the image and filter kernels are discrete quantities.
$S(x,y)=\sum_{-n}^{n}\sum_{-n}^{n}G_{\sigma}(u,v)I(x-u,y-v)dudv$
For acceptable accuracy, we usually retain only Gaussian values that are more than say 0.001 of the peak value. This gives us $2n+1$ elements.

2D convolution can be split into 2 1D convolutions, allowing for a saving of $(2n+1)^2/2(2n+1)$

Differentiation is also computed discretely. We use the central limit theorem: $\frac{\partial S}{\partial x}\approx \frac{S(x+1,y)-S(x-1,y)}{2}$
This is equivalent to convolving with the kernel $[1/2, 0, -1/2]$
It is in this order as it gets flipped in the convolution process.

**Corners**

Edges aren't great when working with images at different angles or scales. They suffer from the aperture problem: when viewing a moving edge, it's only possible to measure motion normal to the edge locally.
Corners do not have this issue so are very valuable to us.

We want to find corners. This can be done by comparing adjacent regions of pixels to a given region of pixels and determining the similarity, or the cross-correlation function. An edge will have a distinct peak, as it's not like anything else around.

Methods of measuring similarity between matrices:
1) Finding the sum of squared differences, essentially the Euclidian distance squared
2) Subtract the mean and normalise, then find the dot product between the two

A blob is an area of uniform intensity in the image. Blob detection is ideal for fixing the location of an object in a scene. Larger objects become more like blobs at larger values of $\sigma$

Convolution with the Laplacian of a Gaussian (LoG) kernel produces a band-pass filtered output. This is because we get $-\omega ^2 G(j\omega)$ which is band-pass. If ever asked about filters, always think about the transform

We can see two Gaussian derivatives as we have convolved with essentially 2 step function. When the scale $\sigma$ is matched to the size of the pulse/blob there is a strong response. 

This response falls with increasing scale so we actually convolve with $\sigma^2 \nabla ^2 G_{\sigma}$ 

In 2D we localise the centre of a blob. The radius of the blob is $\sqrt{2}\sigma$
To help find the exact location and scale efficiently, a set of discrete scales are sampled from an Image Period.

We achieve scale independence by looking at the different resolutions (low-pass filtered at different scales). There are an infinite number of possible resolutions for any image, a three-dimensional function of intensity over location and scale known as the scale space of the image.
$S(x,y,\sigma)=G(x,y,\sigma)*I(x,y)$

Sample this space by making a discrete set of these low-pass filtered images with a logarithmic scale satisfying $\sigma_i = 2^{i/s}\sigma_0$. It doubles after s intervals
$\sigma_{i+1}=2^{1/s}\sigma_{i}$

Blurring with large scales is avoided in 2 ways: subsampling the image after every scale double, and by using a finite set of incremental Gaussian blurs. The result is an image pyramid, with scales in an octave, then subsample the image for the next octave. 

We know that because we have removed high frequencies, we can still sample and perfectly reconstruct our image afterwards.

We know that $G(\sigma_1)*G(\sigma_2)=G(\sqrt{\sigma_1^2+\sigma_2^2})$
Therefore our small incremental blur within octaves takes the form $\sigma_{k_i}=\sqrt{\sigma_{i+1}^2-\sigma_i^2}$
We also want sigma to double after each interval s $\sigma_i = 2^{i/2}\sigma_0$
Combing these two gives: $\sigma_{k_i}=\sigma_i\sqrt{2^{2/s}-1}$

The "s" distinct and small incremental Gaussian filters $\sigma_{k_i}$ need only be computed once and can be reused in each subsequent octave.

Using the Laplace of the Gaussian is quite computationally expensive, so we can approximate it quite accurately using the difference of two Gaussians, giving us a bandpass and a blob detector.
$G(x,y,k\sigma)-G(x,y,\sigma)\approx (k-1) \sigma^2 \nabla^2 G(x,y,\sigma)$
Therefore instead of finding the Laplacian of the Gaussian of the image, we can just take the difference of two levels of blur of the image in the image pyramid. To do this, you need a more blurred image of the same resolution.

For each pixel, to be sure that it is a blob centre at the right scale, we must compare its Difference of Gaussian response to that of the 26 surrounding pixels (8 on the same sigma level, 9 above and 9 below)

The normalisation of scale of this blob has already been done by the scale space and image period, so we just take a 16x16 grid of pixels around our blob centre. Now it is invariant in scale.

To make this invariant in orientation too, we compute the gradients (orientation and magnitude) for each pixel, essentially the edge orientations, and find the keypoint's dominant orientation by looking at the histogram of oriented gradients in the 16x16 grid. We then resample at this dominant orientation.

We can then compute an appropriate descriptor for each feature by either raw or normalised intensities, edges or the SIFT (Scale Invariant Feature Transform)
The simplest would be to dot product the 16x16 grid of intensities, ensuring that you remove the mean and normalise by $\sigma$. However this is very sensitive to pixels shuffling or very bright spots.

A better method would be to make a histogram of all the gradients of each pixel in the 16x16 grid.

**SIFT**
An $N\times N$ patch (typically $N=16$) is extracted at the scale of interest (so that the area is corresponds to never changes). This is then split into $c$ cells, where each has an intensity gradient histogram, weighted by their magnitude and a gaussian window which weights the inner pixels more, to avoid occlusion problems.

If the bins are centred on d directions of c cells, then the resulting descriptor is a $d\times c$ vector. We then normalise, then make sure all values are below 0.2 to remove the dominance of any single element, then renormalise.

This process fails when the 16x16 grid has background and foreground, the background changing drastically with perspective. It also fails when there are repeating patterns, so may match the wrong points. 

To match features, take the Euclidian distance for all the features in the other image. Find the 1st and 2nd largest. Only accept as a match if the ratio of the two is less than 0.7.

To find this correspondence we need a very efficient data structure called a k-d tree.

To form feature maps we can convolve with kernels containing edges, blobs, bars or bright spots.
