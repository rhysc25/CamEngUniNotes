A single neuron takes in a vector of inputs $\textbf{z}$ and computes the dot product with the weights vector $\textbf{w}$ to give the activity $a$. $a=\textbf{w}^T\textbf{z}=\sum_0^D w_d z_d$
The bias $w_0$ is included here as it's implicit that $z_0$ is 1

We then pass the activity into a non-linear activation function, a common one being the sigmoid function $x(a)=\frac{1}{1+exp(-a)}$

Changing the direction of the weight vector changes the direction of the sigmoid function. Increasing the magnitude of the weight vector increases the sharpness of the step.

The goal of training is to vary the weights so that this function outputs almost 1 for one class and almost 0 for another.
The classes are denoted by $y^{(n)}$

We can consider a loss function which looks like this: $G(\textbf{w})=-\sum_n [y^{(n)}log x(\textbf{z}^{(n)};\textbf{w})+(1-y^{(n)})log(1-x(\textbf{z}^{(n)};\textbf{w}))]$

In information theory the surprise of an outcome is $-log$ p(outcome) when observing $y^{(n)}$
This is also the relative entropy between $x(\textbf{z}^{(n)};\textbf{w})$ and $y^{(n)}$

We want to choose the weight which minimises $G(w)$ so we need to use gradient descent.
We pick out initial weight randomly from a Gaussian distribution. Then we apply $w_{i+1}=w_i-\eta \frac{\partial G(w)}{\partial w}|_w$ 
$\eta$ is the learning rate, and says how large steps to take in the direction opposite to the gradient.
Too large, may diverge. Too small, too many iterations.

$\frac{dG(w)}{dw}=\sum_n \frac{dG(w)}{dx^{(n)}}\frac{dx^{(n)}}{da_n}\frac{da_n}{dw_k}$
It's not too hard to show that this is $\frac{dG(w)}{dw}=-\sum_n(y^{(n)}-x^{(n)})z^{(n)}$
However this will just continue increasing the magnitude of the weights until we have essentially a step function, completely confident on the training data but much too overfit.

For this reason we add a regulariser term to our cost function which penalises large weights
$M(w)=G(w)+\alpha E(w)$ where $E(w)=\frac{1}{2}\sum_i w_i^2$
So $\frac{dM(w)}{dw}=-\sum_n(y^{(n)}-x^{(n)})z^{(n)}+\alpha w$
We call this term weight decay

We can interpret the output of the network as a probability that the class is y given an input z
Therefore, assuming the inputs are IID, we can write our cost function as 
$\prod_{n=1}^{N} x_n^{y_n} (1 - x_n)^{1 - y_n}$

As we don't really want to be multiplying small numbers as that can lead to numerical underflow we write then as log probabilities. It can then be shown very easily that this is just $-G(w)$

Let's imagine building up the neural network so we have a single hidden layer. 
We now have an output weights vector with dimensionality $K$ and an input weights matrix with dimensionality $K\times D$
If K is large enough we can now approximate any function.

To perform gradient descent we need $\frac{dG(W,w)}{dW_{ij}}$ which we find through backpropagation
$\frac{dG(W,w)}{dW_{ij}}=\sum_n \frac{dG(W,w)}{dx^{(n)}}\frac{dx^{(n)}}{da^{(n)}}\frac{da^{(n)}}{dx_i^{(n)}}\frac{dx_i^{(n)}}{da_i^{(n)}}\frac{da_i^{(n)}}{dW_{ij}}$

Multi layer perceptrons (MLP) is just a system with more hidden layers, and is more efficient than just one hidden layer with many neurons.

If we want more than 2 classes, so I classes, we make a target vector where the element corresponding to the class is 1, and all the others are zero. This is called one-hot encoding.
We can use this vector to pick out the right probability element.
$G(w)=-\sum_{n}\sum_i y_i^{(n)}\log x_i^{(n)}$

Why do we use hierarchical multi-layered models i.e. deep learning?
Visual scenes are hierarchically organised. Biological vision is hierarchically organised.
If we used an MLP on an image the number of parameters would be $3D^2+D$ where $D$ is the dimension of the image. This is way too large, would take too long to train.

Instead we use convolutional neural networks. High level ideas:
Build in translational invariance, tie weights together
Allow local connectivity only with the use of kernels, weights outside kernel are zero
Subsample more and more up the hierarchy 

Building blocks:
We input the image in native 2D form
We apply the convolutional stage, a kernel representing a feature. $a_{i,j}=\sum_{k,l}w_{k,l}z_{i-k,j-l}$
This gives is a 2D feature map
We then apply the non-linear stage, applying pointwise transformation to the points.
$y_{i,j}=f(a_{i,j})$. A common non-linear function to use here is ReLU, or rectified linear, so is linear for $a>0$ but $0$ for $a<0$
We then implement a pooling stage where we take the max of a $\tau \times \tau$ section, which has the effect of subsampling

The number of parameters now is $3K^2+3\times3K^2+3\times 3K^2+3(D/S)^2$ for a network with 3 convolutional layers
K is the dimension of the kernel, D is the dimension of the image and S is the subsampling fraction.

Each kernel we apply is sensitive to a different type of feature.

Stride length - the jumps you take when moving the grid you pool from
L2 regularisation - the penalising sum of squares of the filters
Input depth - the number of colour channels

Training loss will always decrease but may be because of overfitting. For this reason we have a separate data set called the validation set, and we should stop training when the validation loss starts increasing.

Optimising cross entropy/ maximum log likelihood requires huge computation for images.
For this reason we do stochastic gradient descent.
We randomly pick $m$ images to get a good idea of gradient direction.
Now $G(w)=\frac{N}{M}\sum l(y^{(m)},x^{(m)})$ where $l$ is the cross entropy

The N/M term is necessary as it takes the average.

We can also say that $\frac{dG(w)}{dw}=\frac{N}{M}\sum \frac{d[l(y^{(m)},x^{(m)})]}{dw}$
Because the random nature of this will cause our loss to jump around, we should reduce our learning rate as learning progresses.

Take care to initialise scales of the weights correctly, such that they don't land in the saturation regions of the activation function. For this you could also use non-saturating linear units like ReLU
Also consider data augmentation - include shifted, rotated, cropped versions in the training data.
